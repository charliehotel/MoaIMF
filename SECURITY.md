# 보안 정책

## 지원 범위

현재 개발 중인 `main` 브랜치와 최신 소스 빌드를 대상으로 보안 보고를 받습니다. 서명·공증된 배포 바이너리는 아직 제공하지 않습니다.

## 비공개 보고

다음 문제는 공개 이슈를 만들지 말고 GitHub 저장소의 비공개 Security Advisory로 보고해 주세요.

- 기존 파일 또는 폴더를 덮어쓰거나 삭제하는 동작
- 앱 샌드박스 또는 사용자가 승인한 보안 범위 밖의 접근
- `..`, 잘못된 북마크 또는 링크를 통한 임의 경로 순회
- 심볼릭 링크 대상이나 패키지 내부의 의도하지 않은 변경
- 사용자 승인 없이 감시 폴더가 추가되거나 권한이 유지되는 문제
- 의도하지 않은 네트워크 연결, 텔레메트리 또는 데이터 전송
- 복구 저널 실패로 원래 이름을 복원할 수 없는 문제

보고에는 macOS 버전, 재현 절차, 예상/실제 결과, 영향을 받은 파일시스템 종류와 최소 fixture를 포함해 주세요. 실제 개인정보나 민감한 파일은 첨부하지 마세요.

일반 버그와 기능 요청은 보안 보고가 아니라 저장소 이슈를 사용합니다.
