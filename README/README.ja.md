## 概要

MoaIMF は、ユーザーが選んだフォルダ内のファイル名とフォルダ名を Unicode NFC に正規化する macOS メニューバーアプリです。名前は、ハングル音節の初声(Initial)、中声(Medial)、終声(Final)を集めて合成済み(Composed)の文字にする、という意味です。

macOS では、ファイルシステム、アプリ、ダウンロードツール、展開ツール、外部ストレージ、NAS、クラウド同期ツールを経由するうちに、韓国語ファイル名が NFD のような分解形で保存されることがあります。その場合、Finder では `한글.txt` に見えても、Alfred、ターミナル検索、一部の自動化スクリプトでは `ㅎㅏㄴㄱㅡㄹ.txt` として扱われ、ファイルが見つからないことがあります。

MoaIMF はこの問題を一度だけ実行するクリーンアップスクリプトとして扱いません。ユーザーが許可したフォルダを継続的に監視し、新しく作成またはダウンロードされたファイル名の問題を解消するローカル専用ユーティリティです。

### 監視フォルダ設定

<kbd><img src="../Screenshots/MoaIMF_monitoring_folders_en.png" alt="監視フォルダ設定" width="100%"></kbd>

監視フォルダ設定は、初期値として `Downloads` フォルダから始まります。`+` と `-` ボタンで監視フォルダを追加または削除できます。各フォルダは個別に有効化・無効化でき、権限が切れたフォルダは再選択できます。

フォルダごとの `Scan Now` を押すと、そのフォルダをすぐにスキャンします。結果には NFC 項目、NFD 候補、衝突、保留項目などが表示されるため、何が確認されたかを把握できます。

### ダウンロード安定化の例外

<kbd><img src="../Screenshots/MoaIMF_exceptions_en.png" alt="ダウンロード安定化の例外" width="100%"></kbd>

ダウンロード中のファイルは、名前がまだ確定していなかったり、ファイルサイズや更新時刻が変化し続けていたりします。MoaIMF は `.crdownload`、`.download`、`.part`、`.partial`、`.tmp` などの基本ルールをロックされたルールとして提供し、ユーザー独自の例外ルールも追加できます。

対応するユーザールールは次のとおりです。

- 完全一致のファイル名
- 接尾辞または拡張子
- 最後のパス構成要素に対する `*` と `?` のグロブ

例外ルールに一致した項目は、ルールが削除されるか項目が消えるまで処理されません。例外項目を含む親フォルダも保留されるため、ダウンロード中のフォルダが早すぎる段階でリネームされることを防ぎます。

### 最近の履歴

<kbd><img src="../Screenshots/MoaIMF_recent_history_en.png" alt="最近の履歴" width="100%"></kbd>

最近の履歴は、今日、7日、30日、全期間で表示でき、変更、衝突、権限、エラーの種類で絞り込めます。検索欄はファイルパス、処理理由、イベントタイトル、ルート識別子を対象にし、NFC/NFD の表記差も同じ入力として扱うために正規化した候補も比較します。

履歴はローカルの JSONL ファイルに保存されます。

```text
~/Library/Containers/<app bundle identifier>/Data/Library/Application Support/MoaIMF/history.jsonl
```

履歴は既定で 30 日間保持されます。新しい履歴を追加するとき、保持期間を過ぎた項目は整理されます。

### アプリ情報

<kbd><img src="../Screenshots/MoaIMF_about_en.png" alt="MoaIMF について" width="100%"></kbd>

About ウィンドウには、アプリ名、バージョン、短い説明、著作権表示が表示されます。上部の画像は、`ㅎㅏㄴ -> 한` のように分解された字母が合成済み文字になる概念を示しています。

## 主な機能

- メニューバーで監視状態を確認し、一時停止、再開、終了を実行
- 既定の監視場所として Downloads を使用
- `+` と `-` で複数の監視フォルダを追加・削除
- 監視フォルダとサブフォルダを再帰的に検査
- ユーザーが選んだフォルダだけにセキュリティスコープ付きブックマークでアクセス
- FSEvents で新規ファイルや変更されたパスを検出
- ファイルサイズと更新時刻が安定してから処理
- ダウンロード中のファイルとユーザー定義の例外ルールを保留
- ファイル名とフォルダ名を Unicode NFC に正規化
- 衝突の可能性がある場合は自動上書きしない
- 直接 rename の検証に失敗した場合は復旧可能な一時名戦略を使用
- 変更、衝突、権限、切断、ファイルシステムエラーを最近の履歴に保存
- 今日修正されたファイル数をメニューに表示し、最近の履歴へ移動
- 初回起動時にメニューバーアイコンを見つけるための案内を表示
- ログイン時に起動する登録状態を表示
- アプリ内で表示言語を選択
- 外部サーバー通信、アカウントログイン、テレメトリなし

## 仕組み

MoaIMF はファイルの内容を変更しません。扱うのはファイル名とフォルダ名の Unicode 正規化形式だけです。

処理の流れは次のとおりです。

1. ユーザーが監視フォルダを選択します。
2. アプリはそのフォルダへのアクセス権をセキュリティスコープ付きブックマークとして保存します。
3. FSEvents が新規ファイル、新規フォルダ、名前変更、ダウンロード完了などの変化を通知します。
4. スキャンサービスが対象パスと下位項目を検査します。
5. 例外ルール、パッケージ、シンボリックリンクの対象、権限問題、ファイル安定化状態を確認します。
6. NFD など NFC ではない名前を見つけると、NFC の目標名を計算します。
7. 同じフォルダ内で正規化後に同じ名前になる項目がないか衝突を確認します。
8. 衝突がなく安定した項目だけが名前変更候補になります。
9. 変更直前にファイルの同一性を再確認し、rename を実行します。
10. 変更後、実際のファイル名が NFC バイトとして保持されたかを検証します。
11. 結果を最近の履歴に保存し、メニューの今日の修正数を更新します。

この構造により、MoaIMF は衝突が疑われるファイルを勝手にマージせず、`-1`、`copy`、`복사본` のような新しい名前も自動作成しません。ユーザーの判断が必要な状況は、最近の履歴と通知に残します。

## 使い方

### 1. アプリを起動

`MoaIMF.app` を起動すると、通常のアプリのように Dock に長く残るのではなく、メニューバーにアイコンが表示されます。メニューバーのアイコンが多い場合や MacBook のノッチに隠れる可能性があるため、初回起動時には案内ウィンドウを表示します。

### 2. 監視フォルダを設定

メニューから `Watched Folder Settings...` を開きます。既定のフォルダは Downloads です。必要に応じて `+` ボタンで別のフォルダを追加します。

推奨される使い方は次のとおりです。

- Downloads のように新しいファイルがよく作られる場所を監視対象に登録します。
- プロジェクトソース、写真ライブラリ、アプリバンドルなど構造が繊細な場所は必要なときだけ登録します。
- 外部ディスクや NAS 同期フォルダは、まず少量のファイルで試します。

### 3. 既存項目の扱いを選択

初めてフォルダを追加するとき、既存項目もすぐに正規化するか、今後作成される項目だけを監視するかを選べます。

- `Normalize Existing Items`: 現在のフォルダ内にすでにある Non-NFC 名も検査し、承認された候補を変更します。
- `Watch New Items Only`: 既存のファイル名はそのままにし、以後新しく作成または変更された項目から処理します。

既存ファイルが多いフォルダでは、先にスキャン結果を確認してから処理する方が安全です。

### 4. 一時停止と再開

メニューの `Pause Watching` を選ぶと、監視と自動リネームが止まります。設定、履歴、フォルダ一覧は保持されます。一時停止中はメニューバーアイコンのアニメーションも止まり、初期フレームを表示します。

再び処理するには `Resume Watching` を選びます。

### 5. すぐにスキャン

`Scan All Now` は登録済みの監視フォルダ全体を再スキャンします。監視フォルダ設定画面の各行にある `Scan Now` は、そのフォルダだけを検査します。

手動スキャンは次のような場合に便利です。

- アプリが終了している間にファイルが追加された場合
- 外部ディスクを再接続した場合
- 例外ルールを変更したあと、保留項目を再確認したい場合
- 最近の履歴と実際のフォルダ状態を比較したい場合

### 6. 言語を変更

メニューの `Language` からアプリの表示言語を直接選べます。現在手動で選択できる言語は次のとおりです。

- System Default
- English
- 한국어
- 日本語
- 简体中文
- 繁體中文
- Tiếng Việt
- Français
- Deutsch
- Español
- Português
- ไทย
- العربية

提供される言語は利便性のための AI 翻訳です。誤訳や追加してほしい言語があれば、`Issues` で知らせてください。

### 7. ログイン時に起動

メニューで `Launch at Login` をオンにすると、MoaIMF を macOS のログイン項目に登録します。メニューには登録状態も表示されます。macOS がユーザー承認を求める場合は、システム設定を開いて承認してください。

### 8. 終了

`Quit MoaIMF` を選ぶと、監視作業とセキュリティスコープアクセスを整理してからアプリを終了します。別の daemon や helper プロセスは残しません。

## インストールとビルド

現在はソースからのビルドを前提としています。Developer ID 署名と Apple 公証を済ませた配布ファイルはまだ提供していません。

### 要件

- macOS 13 Ventura 以降
- Xcode 16 以降、または互換性のある Xcode Command Line Tools
- Swift 6 toolchain
- Git

ビルドスクリプトは macOS 標準ツールも使用します。

- `swift`
- `xcrun swift-format`
- `sips`
- `python3`
- `codesign`
- `ditto`

実行時に別途サーバー、データベース、ネットワーク API は必要ありません。

### リポジトリを取得

```sh
git clone https://github.com/charliehotel/MoaIMF.git
cd MoaIMF
```

リポジトリ URL は実際の公開場所に合わせて変更されることがあります。

### 検査とビルド

全体の検査、テスト、アプリバンドル生成をまとめて実行するには、次のコマンドを使います。

```sh
scripts/check.sh
```

アプリバンドルだけを作るには、次のコマンドを使います。

```sh
scripts/build-app.sh
```

生成されたアプリは次の場所に作られます。

```text
.build/MoaIMF.app
```

ローカルで実行するには:

```sh
open .build/MoaIMF.app
```

`scripts/build-app.sh` が作成するアプリは、ローカルテスト用の ad-hoc 署名アプリです。他の Mac に配布するには、Developer ID 署名と notarization を別途設定する必要があります。

## ローカルデータの場所

MoaIMF はアプリ状態と履歴を、macOS アプリサンドボックスコンテナ内の Application Support に保存します。

```text
~/Library/Containers/<app bundle identifier>/Data/Library/Application Support/MoaIMF/
```

`<app bundle identifier>` は、インストールされたアプリビルドに含まれるバンドル識別子によって変わります。

主なファイルは次のとおりです。

| ファイル | 説明 |
|---|---|
| `watched-folders.json` | 監視フォルダ一覧、有効状態、セキュリティスコープ付きブックマーク |
| `stability-rules.json` | ダウンロード安定化の基本ルールとユーザー例外ルール |
| `history.jsonl` | 最近の処理履歴 |
| `recovery/` | 名前変更中に失敗した場合の復旧用ジャーナル |

一時停止状態、言語選択、初回案内を非表示にするかどうかなど、一部のアプリ設定は `UserDefaults` にも保存されます。

## 安全性とプライバシー

MoaIMF の基本原則は「安全に名前だけを変更し、分からない場合は止まる」です。

- ファイル内容は読み取りも変更もしません。
- ユーザーが選んだフォルダだけにアクセスします。
- セキュリティスコープ付きブックマークで macOS の権限モデルに従います。
- `.app`、`.bundle`、`.framework`、`.photoslibrary` などのパッケージ内部は検査しません。
- シンボリックリンクの対象はたどりません。
- 正規化後に同じ名前になる兄弟項目がある場合は変更しません。
- 名前変更の前後でファイルの同一性を検証します。
- 検証済みの NFC 名をファイルシステムが保持しない場合、その状況をエラーとして記録します。
- すべての処理はローカルで行われます。
- ネットワーク通信、アカウントログイン、分析イベント、テレメトリはありません。

## 制限事項

- MoaIMF は macOS 全体のファイル名保存方式を変更しません。
- すべてのアプリがファイル名を NFC で保存するよう強制しません。
- 衝突ファイルを自動でマージしたり、新しい名前で解決したりしません。
- Spotlight や Alfred のインデックスを直接再生成しません。
- ファイルシステム、同期ツール、外部ストレージが名前のバイト列を再び変更する場合、すべての状況を自動復旧できるわけではありません。
- 現在はソースビルドとローカルテスト用アプリバンドルを中心に動作します。
- 自動更新、App Store 配布、公証済み DMG 配布はまだありません。

## 削除

1. メニューで `Launch at Login` をオフにします。
2. メニューで `Quit MoaIMF` を選びます。
3. インストールした `MoaIMF.app` を削除します。
4. ローカル状態も削除する場合は、アプリコンテナ内の MoaIMF Application Support フォルダを削除します。

```text
~/Library/Containers/<app bundle identifier>/Data/Library/Application Support/MoaIMF/
```

`<app bundle identifier>` は、インストールされたアプリビルドに含まれるバンドル識別子によって変わります。

この操作は、すでに NFC に変更されたファイル名を NFD に戻しません。

## 開発

このプロジェクトは Swift Package Manager ベースです。

```text
Sources/
  MoaIMFCore/   正規化、スキャン、監視、ストア、安全な rename エンジン
  MoaIMFUI/     AppController、メニュー、設定、履歴、About、ローカライゼーション
  MoaIMFApp/    macOS アプリのエントリポイントとメニューバーラベル
Tests/
  MoaIMFCoreTests/
  MoaIMFUITests/
```

よく使うコマンドは次のとおりです。

```sh
xcrun swift-format lint --strict --recursive Sources Tests Package.swift
swift test
swift build
scripts/build-app.sh
```

sandbox やキャッシュ権限の問題を減らすには `scripts/check.sh` を使うのがよいです。このスクリプトは SwiftPM キャッシュとモジュールキャッシュを `.build/` 以下に指定します。
