# MoaIMF v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and publish a native macOS menu bar app that recursively watches user-approved folders and safely normalizes stable file and folder names to Unicode NFC.

**Architecture:** Use a dependency-free Swift Package with three production targets: `MoaIMFCore` for filesystem logic, `MoaIMFUI` for application state and SwiftUI views, and `MoaIMFApp` for the `@main` entry point. FSEvents only schedules reconciliation scans; an actor serializes scans, stability checks, collision analysis, bottom-up renames, history writes, and cancellation. A reproducible shell script assembles and ad-hoc signs the `.app` bundle without an external project generator.

**Tech Stack:** Swift 6 language mode, SwiftUI `MenuBarExtra`, Foundation, CoreServices FSEvents, ServiceManagement `SMAppService`, UserNotifications, XCTest, Swift Package Manager, `swift-format`, GitHub Actions.

**Source Spec:** `docs/superpowers/specs/2026-06-21-moaimf-v0.1-design.md`

**Verified Local Toolchain:** Swift 6.3.2, Xcode 26.5, macOS 26.5.1, `swift-format` 6.3.0 on Apple Silicon.

**Primary API References:** [MenuBarExtra](https://developer.apple.com/documentation/swiftui/menubarextra), [SMAppService](https://developer.apple.com/documentation/servicemanagement/smappservice), [FSEventStreamCreate](https://developer.apple.com/documentation/coreservices/1443980-fseventstreamcreate), [App Sandbox file access](https://developer.apple.com/documentation/security/app_sandbox/accessing_files_from_the_macos_app_sandbox), [GitHub macOS 15 runner image](https://github.com/actions/runner-images/blob/main/images/macos/macos-15-Readme.md).

## Implementation Rules

- Follow strict Swift 6 concurrency. UI state is `@MainActor`; mutable background state lives in actors or a private serial queue.
- Do not use `try!`, force casts, force unwraps, `fatalError`, or silent `catch` blocks in production code.
- Do not use path existence as the normalization collision test. Compare raw sibling names grouped by their target normalization key.
- Never follow symlink targets or enumerate package contents.
- Every filesystem mutation must have a preflight check, a recorded outcome, and a postcondition check.
- Tests use temporary directories and never mutate the user's real Downloads folder.
- Run `scripts/check.sh` before each commit once that script exists.

```markdown
# MoaIMF Design System

## Product character
Quiet, trustworthy, native, and reversible. MoaIMF must look like a macOS system utility rather than a dashboard or branded web app.

## Platform language
- Use SwiftUI system controls and SF Symbols.
- Use SF Pro through system text styles; never bundle a custom font.
- Respect light mode, dark mode, Increase Contrast, Reduce Motion, and keyboard focus.

## Color
- Use semantic system colors only: primary, secondary, accent, green, orange, red, and separator.
- Green means actively watching, orange means paused or disconnected, and red means an unresolved collision or permission failure.
- Never rely on color without a symbol and text label.

## Typography
- Menu title: headline.
- Primary status: body with semibold emphasis.
- Paths and timestamps: caption using monospaced digits only where alignment matters.
- Destructive and warning copy must remain plain and specific.

## Spacing and geometry
- Base spacing unit: 8 points.
- Compact row gaps: 4 points; section gaps: 16 points; window padding: 20 points.
- Use system control sizes and system corner radii. Do not place every section in a card.

## Components
- MenuBarExtra uses menu style for status and frequent commands.
- Settings uses a native list with folder icon, name, path, status, and per-folder actions.
- Plus and minus buttons follow the standard macOS list editing pattern.
- Alerts state what was not changed and why; no confirmation is required for safe, non-mutating scans.

## States
- Watching: checkmark.circle.
- Scanning: arrow.triangle.2.circlepath.
- Paused: pause.circle.
- Attention: exclamationmark.triangle.
- Empty, loading, permission, disconnected-volume, collision, and unsupported-filesystem states must each have visible text.
```

- [x] **Step 3: Implement rename preconditions and direct path**

`SafeRenameEngine.apply(_:)` must:

1. never accept collision participants as rename candidates;
2. sort the collision-free candidates by path depth descending;
3. re-read raw siblings and recompute the target group immediately before mutation;
4. confirm the source identity still matches;
5. call `FileManager.moveItem(at:to:)` for the direct NFD-to-NFC rename;
6. re-read the parent directory and verify the stored raw name is NFC and identity is unchanged;
7. return one `RenameOutcome` per candidate instead of swallowing errors.

- [x] **Step 4: Implement two-step fallback and startup recovery**

Fallback uses a sibling name formatted as `.moaimf-<operation UUID>-temporary`, records `planned` before mutation, records `temporary` after the first move, and uses an exclusive final move. If the final move or verification fails, restore the original raw name. At startup, replay any `temporary` record by preferring restoration unless the verified NFC target already owns the original identity.

Implement the exclusive final move with Darwin rather than a check-then-move race:

```swift
import Darwin
import Foundation

private func moveExclusively(from source: URL, to target: URL) throws {
    try source.withUnsafeFileSystemRepresentation { sourcePath in
        guard let sourcePath else { throw POSIXError(.EINVAL) }
        try target.withUnsafeFileSystemRepresentation { targetPath in
            guard let targetPath else { throw POSIXError(.EINVAL) }
            guard renameatx_np(
                AT_FDCWD,
                sourcePath,
                AT_FDCWD,
                targetPath,
                UInt32(RENAME_EXCL)
            ) == 0 else {
                throw POSIXError(POSIXErrorCode(rawValue: errno) ?? .EIO)
            }
        }
    }
}
```

No temporary name may be left without an active journal record.

```swift
private func globMatches(pattern: [Character], value: [Character]) -> Bool {
    var matches = Array(
        repeating: Array(repeating: false, count: value.count + 1),
        count: pattern.count + 1
    )
    matches[0][0] = true

    if !pattern.isEmpty {
        for patternIndex in 1...pattern.count where pattern[patternIndex - 1] == "*" {
            matches[patternIndex][0] = matches[patternIndex - 1][0]
        }
    }

    if !pattern.isEmpty, !value.isEmpty {
        for patternIndex in 1...pattern.count {
            for valueIndex in 1...value.count {
                switch pattern[patternIndex - 1] {
                case "*":
                    matches[patternIndex][valueIndex] =
                        matches[patternIndex - 1][valueIndex]
                        || matches[patternIndex][valueIndex - 1]
                case "?":
                    matches[patternIndex][valueIndex] =
                        matches[patternIndex - 1][valueIndex - 1]
                default:
                    matches[patternIndex][valueIndex] =
                        matches[patternIndex - 1][valueIndex - 1]
                        && pattern[patternIndex - 1] == value[valueIndex - 1]
                }
            }
        }
    }

    return matches[pattern.count][value.count]
}
```
